Project 1 Summary

Penetration Testing – Part 1

Reconnaissance

Gathered information about the target using various methods:

  • WHOIS Lookup: Retrieved domain registration details such as registrar and owner information.
  • DNS Footprinting: Collected DNS records like A, MX, and TXT using tools such as nslookup and dig.
  • Network Footprinting: Mapped the network using tools like Traceroute and Nmap.
  • Social Engineering: Leveraged publicly available information from social media and forums to understand potential weaknesses.

Port Scanning

Identified open ports and associated services:

  • Used nmap to perform scans and determine services running on specific ports (e.g., SSH, FTP).
  • Discovered MSSQL instances using nmap scripting features.
  • Identified and documented services such as SSH, FTP, and SNMP.

Vulnerability Scanning

Conducted vulnerability analysis using multiple tools:

  • SMB Login Check: Tested for weak credentials on SMB shares using smbclient.
  • VNC Authentication: Checked for vulnerabilities in VNC services to exploit unauthorized access.
  • WMAP Web Scanner: Analyzed web applications for vulnerabilities.
  • Nessus: Ran a comprehensive vulnerability scan, identifying misconfigurations and missing patches.

Conclusion

The reconnaissance phase provided critical insights into the target, enabling focused scanning and vulnerability identification. Open ports and vulnerable services were documented, forming the foundation for further penetration testing in Project 2.

Project 2 Summary

Penetration Testing – Part 2

Client-Side Attacks

Performed various client-side attack techniques, including:

  • Binary Payloads: Created and deployed executable payloads to exploit vulnerabilities.
  • Client-Side Exploits: Exploited client-side vulnerabilities in applications such as browsers and email clients.
  • Social Engineering Toolkit: Used tools like the Social Engineering Toolkit (SET) to simulate phishing attacks and gather credentials.
  • Privilege Escalation: Gained elevated privileges through misconfigurations or vulnerabilities.

Post-Exploitation Techniques

Executed advanced post-exploitation tasks to control the system and extract critical information:

  • Privilege Escalation: Identified and exploited privilege escalation vulnerabilities.
  • Pass the Hash: Used hashed credentials to access other systems without cracking passwords.
  • Clear Event Management: Cleared event logs to remove traces of exploitation.
  • Interacting with the Registry: Accessed and modified registry keys to maintain persistence.
  • Packet Sniffing: Captured network traffic to analyze sensitive data in transit.
  • Screen Capture: Took screenshots to monitor user activity.
  • Searching for Content: Searched for sensitive files and information on the compromised system.
  • Time Stamp Modification: Altered timestamps to obscure activity logs.
  • John the Ripper: Used John the Ripper for password cracking and analysis.

Maintaining Access

Implemented techniques to ensure continued access to the system:

  • Keylogging: Monitored and captured keystrokes to gather credentials and monitor user activity.
  • Persistent Backdoors: Installed backdoors for long-term access to the system.

Conclusion

The techniques used in client-side attacks, post-exploitation, and access maintenance were critical in demonstrating how attackers can exploit and control systems. These tasks complement the reconnaissance and vulnerability scanning conducted in Project 1 to form a comprehensive penetration testing methodology.
